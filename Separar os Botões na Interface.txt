Separar os Botões na Interface

Quando dois ou mais botões são inseridos dentro do mesmo CTkFrame (ou outro widget de contêiner), eles compartilham o fundo do frame, criando uma aparência de continuidade entre eles. Isso pode dar a impressão visual de um único botão, mesmo que sejam elementos distintos. Para resolver esse problema e deixar os botões separados visualmente, você pode posicioná-los diretamente na janela principal (CTk) ou em frames individuais.

Passos Detalhados para Resolver
Remover os Botões do Mesmo Frame:

Em vez de colocar ambos os botões dentro de um único CTkFrame, coloque-os diretamente na janela principal (tela_principal) ou em frames separados.

Espaçamento e Margens:

Use o parâmetro pady (margem vertical) e/ou padx (margem horizontal) para criar espaçamento entre os botões.

Personalização Independente:

Cada botão pode ser configurado com tamanho e posicionamento próprios. Isso ajuda a deixar claro que são elementos separados.





Explicação dos Ajustes:
Configuração Individual dos Botões:

Cada botão agora está diretamente associado à janela principal (tela_principal) como seu contêiner, garantindo que não compartilhem fundos ou estilos de frames.

Espaçamento entre os Botões:

O parâmetro pady é usado para criar separação visual, garantindo que os botões não fiquem agrupados.

Estilo Independente:

Os parâmetros como width (largura) podem ser ajustados para definir dimensões específicas para cada botão.

Cor de Fundo da Janela:

A propriedade fg_color pode ser usada para personalizar a aparência da janela principal. Certifique-se de usar cores válidas como nomes comuns ("white", "red") ou códigos hexadecimais ("#FFFFFF").
